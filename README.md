# zh2-leisu-guide

本仓库用于归档和发布多个独立 HTML 页面。这些页面均为静态内容，不针对任何特定网站或域名，旨在提供一种轻量、自包含的信息呈现方式。

## 项目简介

`zh2-leisu-guide` 是一个以 HTML 页面为基本单元的归档仓库。每个页面在功能上相互独立，内容涵盖多种主题与用途。仓库本身不绑定任何具体平台或服务，仅作为静态文件的存储与分发点。

## 目录说明

```
.
├── index.html          # 入口页面（可选）
├── pages/              # 独立 HTML 页面存放目录
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
└── assets/             # 公共资源（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/` 目录下存放所有独立 HTML 页面，每个页面均可单独访问。
- `assets/` 目录存放页面间共享的样式表、JavaScript 脚本及图片等资源。
- 若存在 `index.html`，通常用作导航或说明页。

## 页面归档说明

- 每个 HTML 页面均为自包含的静态文件，不依赖外部数据库或后端服务。
- 页面内容采用纯前端技术构建，可直接在浏览器中打开使用。
- 页面之间无强依赖关系，可独立更新或删除。
- 鼓励在页面内通过注释或元数据标注版本、创建日期、简要说明等信息。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建对应的 HTML 文件，并视需要更新 `index.html` 中的链接。
- 更新页面：直接修改对应 HTML 文件，注意保持资源引用路径正确。
- 删除页面：移除 `pages/` 下对应文件，并清理 `index.html` 中的相关链接。
- 公共资源修改：若修改 `assets/` 下的共享资源，请确保所有引用该资源的页面兼容。
